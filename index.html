ISMS Scope

Scope-

This ISMS scope covers the following components within my home network and study environment. The ISMS covers the normal operation, access control, data storage, synchronization and account security of the defined environment. The purpose of defining this scope is to clearly identify what systems, devices, and information assets are included and protected under the ISMS.

The scope is limited to systems and services that I personally own, use, and control. Anything outside this control is excluded and justified in the exclusions section.

  • Infrastructure- DNA router, Wi-Fi network
  • Devices- HP windows laptop (primary device for coursework, assignments and technical exercises), iPhone 13 (used for cloud access and MFA), Kindle (used for reading coursework sometimes)
  • Information Assets- Course materials, personal notes, repositories and all other associated accounts, credential and MFA

These assets are required to support my studies and coursework. The security of these assets is important to ensure availability of study materials, protection of personal data, and secure access to cloud-based services.

Key interfaces

Key interfaces describe how the in-scope systems interact with external systems and services. These interfaces represent important security boundaries.

  • Cloud services- GitHub, iCloud, OneDrive
  • Remote access- VPN, SSH
  • External boundary- Router connecting home network to internet
  • Suppliers- ISP (DNA), device vendors and cloud service providers

Access to cloud services is performed over the internet using authenticated user accounts. Remote access technologies such as VPN and SSH are used when required for coursework or technical exercises.

Network and interface diagram

The home network consists of a DNA router that provides a connection between the internal home network and the internet. The HP laptop, iPhone, and Kindle connect to the router using the Wi-Fi network. Cloud services such as GitHub, iCloud, and OneDrive are accessed through the internet connection provided by the router.

Interested parties and ISO 27001 Alignement

Interested parties are individuals or organizations that have an interest in the security of the home network and study environment. Their needs and expectations are considered as part of the ISMS.

As the ISMS owner, I am responsible for maintaining the security of the in-scope assets, accepting residual risks, and ensuring that the ISMS remains suitable for my study environment.
